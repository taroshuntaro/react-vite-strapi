# React (Vite) + Strapi Dev Environment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a working local development environment where React+Vite runs on the host and Strapi+PostgreSQL run in Docker Compose, connected via a Vite dev-server proxy.

**Architecture:** Two independent apps (`frontend/`, `backend/`) in one repo. Frontend runs via `npm run dev` on the host and proxies `/api` and `/uploads` requests to Strapi. Backend (Strapi) and PostgreSQL run as Docker Compose services on the developer's machine. All Strapi secrets and DB credentials are supplied via Compose environment variables, not files baked into the image.

**Tech Stack:** React 19 + Vite (TypeScript template), Strapi 5 (TypeScript), PostgreSQL 17, Docker Compose, npm.

## Global Constraints

- Node.js version: 24.x LTS (pin via `.node-version` and `package.json` `engines.node`)
- Package manager: npm only (`package-lock.json` committed in both `frontend/` and `backend/`)
- Database: PostgreSQL (same product dev/prod), never SQLite
- Strapi version: 5.x, TypeScript template
- Uploads: local `public/uploads` (no S3/MinIO in this plan)
- Scope: local development only — no production Dockerfiles, no CI, no ECS in this plan

### Task 3: Frontend Vite proxy configuration

**Files:**
- Modify: `frontend/vite.config.ts`
- Create: `frontend/.env.development`

**Interfaces:**
- Consumes: nothing from other tasks.
- Produces: proxy behavior later verified in Task 6 — requests to `http://localhost:5173/api/*` and `/uploads/*` are forwarded to `env.DEV_API_PROXY_TARGET` (default `http://localhost:1337`).

- [ ] **Step 1: Replace `frontend/vite.config.ts` contents**

```typescript
import { defineConfig, loadEnv } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd(), "");

  return {
    plugins: [react()],
    server: {
      port: 5173,
      proxy: {
        "/api": {
          target: env.DEV_API_PROXY_TARGET ?? "http://localhost:1337",
          changeOrigin: true,
        },
        "/uploads": {
          target: env.DEV_API_PROXY_TARGET ?? "http://localhost:1337",
          changeOrigin: true,
        },
      },
    },
  };
});
```

- [ ] **Step 2: Create `frontend/.env.development`**

### Task 5: Docker Compose (db + strapi)

**Files:**
- Create: `compose.yaml`

**Interfaces:**
- Consumes: `DATABASE_NAME`, `DATABASE_USERNAME`, `DATABASE_PASSWORD`, `POSTGRES_VERSION`, `APP_KEYS`, `API_TOKEN_SALT`, `ADMIN_JWT_SECRET`, `TRANSFER_TOKEN_SALT`, `JWT_SECRET`, `ENCRYPTION_KEY` from `.env` (Task 1).
- Produces: `db` service reachable at `db:5432` from the `strapi` service; `strapi` service reachable at `http://localhost:1337` from the host.

- [ ] **Step 1: Create `compose.yaml`**

```yaml
services:
  db:
    image: postgres:${POSTGRES_VERSION:-17}-alpine
    environment:
      POSTGRES_DB: ${DATABASE_NAME}
      POSTGRES_USER: ${DATABASE_USERNAME}
      POSTGRES_PASSWORD: ${DATABASE_PASSWORD}
    volumes:
      - postgres-data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DATABASE_USERNAME} -d ${DATABASE_NAME}"]
      interval: 5s
      timeout: 5s
      retries: 10

  strapi:
    image: node:24-alpine
    working_dir: /app
    command: sh -c "npm install && npm run develop"
    environment:
      NODE_ENV: development
      HOST: 0.0.0.0
      PORT: 1337

      DATABASE_CLIENT: postgres
      DATABASE_HOST: db
      DATABASE_PORT: 5432
      DATABASE_NAME: ${DATABASE_NAME}
      DATABASE_USERNAME: ${DATABASE_USERNAME}
      DATABASE_PASSWORD: ${DATABASE_PASSWORD}

      APP_KEYS: ${APP_KEYS}
      API_TOKEN_SALT: ${API_TOKEN_SALT}
      ADMIN_JWT_SECRET: ${ADMIN_JWT_SECRET}
      TRANSFER_TOKEN_SALT: ${TRANSFER_TOKEN_SALT}
      JWT_SECRET: ${JWT_SECRET}
      ENCRYPTION_KEY: ${ENCRYPTION_KEY}
    volumes:
      - ./backend:/app
      - strapi-node-modules:/app/node_modules
    ports:
      - "1337:1337"
    depends_on:
      db:
        condition: service_healthy

volumes:
  postgres-data:
  strapi-node-modules:
```

Note: `strapi-node-modules` is a separate named volume so the container's `node_modules` (built for Linux, including native deps like `sharp`) never gets overwritten by the host bind mount of `./backend`.
